# Build a Personal Cloud Drive (Next.js + Vercel + AWS S3, No Database)

You are a senior full-stack engineer. Build a production-quality personal cloud storage application using **Next.js 15 (App Router)** and **TypeScript**.

## Goal

Create a self-hosted Google Drive / Notion-like application for a **single user**.

The application will be deployed on **Vercel**.

**Do NOT use any database.**

Instead:

* Store all files in AWS S3.
* Store all notes in AWS S3.
* Store all metadata (folders, notes, files, settings) as JSON files inside S3.
* Authentication should be a simple username/password login using environment variables.

The application should be designed so that replacing the JSON metadata layer with a database later requires minimal changes.

```json

  {
    "id":"root",
    "name":"Home",
    "parentId":null,
    "createdAt":""
  }

```

files.json

```json
[
  {
    "id":"",
    "name":"Resume.pdf",
    "folderId":"root",
    "key":"uploads/documents/file.pdf",

    "updatedAt":""
```

# Authentication

Single user only.

Environment variables:

```env
APP_USERNAME=admin
APP_PASSWORD_HASH=<bcrypt hash>

JWT_SECRET=<random>

AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_REGION=
AWS_BUCKET_NAME=
```

Requirements:

* Login page
* Logout
* JWT stored in HTTP-only cookie
* Middleware protects all routes except /login
* Password hashing with bcrypt
* Session expiration
